Vibe coding完了一地鸡毛，有人靠清理AI垃圾代码一周赚7万

昨天在HN上刷到一个帖子，标题给我整不会了。（以后我会像傻逼一样天天刷HN） 「我们一周收1万美元，专门删AI写的垃圾代码。」 三个波兰工程师搭了个叫 Slopfix 的服务，帮人清理 vibe coding 留下的烂摊子。承诺把代码量砍至少50%，达不到按比例退钱。上线几小时，247个赞，评论区直接炸了。 说实话，我看完第一反应是，这特么也太聪明了吧。 Vibe coding火了以后，确实让很多不怎么会写代码的人也能做出东西。「写」这个动作成本一下降到了零。但问题也跟着来了，AI生成的代码，一个月之后还能看懂吗？半年之后还能改吗？？？ 我自己也用Claude Code写东西。说实话，刚开始确实是爽，几句话就把一个功能搞出来了。但两周后想加个新功能，改一行崩三处，给我整不会了。AI不会重构，它只会一直在你已有的代码上叠叠乐，直到叠出一座危楼。 Odra干的事，就是把这栋危楼拆了重盖。三个资深工程师，一周时间，收1万美元。他们的客户群很明确，那些靠AI快速搭起来的产品，现在代码失控了，加功能要三天改bug要五天。 这事给了我一个启发。既然有人敢收1万一周，那普通人能不能也做？我觉得可以。说真的，这里面藏着三个完全不同的赚钱路子。 第一个方向，你说门槛最低。 做AI代码审查。不是去删代码重写，而是在AI写代码的时候就帮人把关。 你想想看，评论区有个老哥，他的活儿就是给一个非技术CEO审AI写的代码。90%的时间在跑代码review，把AI的Claude.md引导到正确的架构上去。不用自己写一行代码，就把人家的技术债扼杀在摇篮里。 你不需要是顶级架构师，有点代码基础、能看懂AI写的代码质量就行。收费可以按月包，一个月几千块，帮小团队盯着AI写的每一行代码。 第二个方向，门槛高一些，但利润也高。 直接复刻Odra的模式，帮人重构AI生成的混乱代码库。定价可以灵活点，不用像Odra那样一次性1万。可以按项目报价，小的几千，大的几万。关键是先把客户找到。什么样的客户？那些用vibe coding做了MVP、现在产品要上线的初创团队。他们的代码库已经到了「不改不行」的阶段。 加上一个我自己的想法，跟做AI代码审计的人不一样，做重构的还需要懂怎么用AI帮自己加快速度。Odra自己也说了，他们也用Claude Code，只是很短的控制链。区别在于30年的工程经验加上AI的效率和速度。说实话，这是最有护城河的方向。 第三个方向，我觉得是未来最值钱的。 帮企业建AI代码的「守门人」体系。CLAUDE.md怎么写才能让AI不乱来、lint规则设哪些才能拦住高频错误、CI/CD里加什么检查能第一时间发现AI生成的坏代码。这些东西本身是可复用的，做完一家可以复制到另一家。 定价可以走年费制，帮企业搭好体系之后每个月维护一下规则库。持续收入，不用每单都从零开始。 我说「理论上」是因为我自己也还在摸索，不是所有东西都跑通了。但这个方向，我是真的觉得，是vibe coding时代最被低估的服务机会。 三个方向对应三种能力，基本的代码判断力、中级的AI重构能力、高级的体系搭建能力。从哪个切入都行，关键是先动起来。 就像我之前聊vibe coding做副业那篇里说的，AI降低的是执行门槛，不是判断门槛。代码AI写起来容易，但判断它写得好不好、值不值得留、怎么改才是人的价值。 磨平一些信息差。 最后补充一句。我知道很多人担心vibe coding会让程序员的技能贬值。但换个角度看看，当每个人都能用AI「写」代码的时候，会「读」代码、会「删」代码的人，反而更值钱了。 我整理了一份AI代码审查和重构的提示词包，包括CLAUDE.md的最佳实践模板、代码审查的Prompt框架、以及重构策略清单。有需要的可以看看👇

关于AI代码清理的常见问题

AI代码清理服务需要什么技术背景？

不需要顶级架构师水平，但需要有基本的代码阅读和判断能力。如果你是程序员，能看懂代码质量好坏，就已经具备入门条件。第一个方向（代码审查）对技术门槛的要求最低，很多人从帮朋友的项目开始，慢慢积累案例。

AI代码清理服务怎么收费？

定价分三档：AI代码审查按月包（3000-8000元/月）、代码重构按项目报价（5000-50000元不等）、AI代码体系搭建走年费制（2-5万/年）。可以参考Odra的模式，承诺量化指标（如代码量减少50%），达不到按比例退钱，降低客户决策门槛。

AI清理代码的市场有多大？

随着vibe coding的普及，AI生成的烂代码正成倍增长。Odra三个工程师一周就能收1万美元，说明市场需求真实存在。而且这个市场会持续扩大——只要AI还在帮人写代码，就会有人需要帮人擦屁股。
