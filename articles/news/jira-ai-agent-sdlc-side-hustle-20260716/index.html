Jira全面拥抱AI Agent开发，项目管理工具开始管AI了

昨天Atlassian发了一篇文章，标题看着挺正经的，但我读完第一段就没绷住。

他们跟DX做了项研究，AI使用量涨了65%，团队交付速度只提升了10%到15%。AI用了这么多，产出没翻倍。问题不在模型写不好代码，而是「软件工程从来不只是写代码」。

你想想看，一个开发团队每天在干嘛。开会对齐需求、翻以前代码看架构、搞清楚哪些约束不能动、做完找人Review、上线盯着有没有问题。这些事，AI agent一个都做不了。代码写得多快，这些环节都会卡住。

这个研究结果挺扎心的。我之前聊AI编程的那篇文章也写过，AI coding agent真正的瓶颈不是代码质量，是「怎么告诉它要做什么」以及「做完怎么知道它做对了」。

好，那Atlassian做了什么。他们发布了Jira的AI原生开发能力，直接把AI agent塞进了企业最常用的项目管理工具里。

Jira这次到底发了什么

最核心的变化就是，Jira现在可以直接给AI coding agent安排任务了。你在Jira里建一个ticket，分配给Claude Code、Cursor、GitHub Copilot（之后还有Codex），agent做完之后直接把PR交回来。整个过程在Jira里就能看到，agent卡在哪了、等谁Review、做完了没有。

他们还做了Jira Coding Agent，一个内置agent，专门干修bug、补测试、更新文档的重复活，直接绑在每个付费Jira计划里。

Jira Planner也很骚，它能从你们代码库、Confluence和Jira历史记录里拉上下文，自动生成一份既给人看也给agent看的技术方案。同一个文档，人读得懂，agent也能执行。

还有Loom视频转任务，你录个屏说一下你要什么，Loom能把它变成结构化的agent指令。这一下给我整不会了，录个视频就能给AI派活，这事半年前谁敢想。

加上DX的AI成本管理，所有agent花的token钱统一在一个仪表盘里看，哪个产出高、哪个浪费钱，一目了然。

这一整套东西放到一块，我感觉比单个新工具发布要重要得多。

这对做副业的普通人藏着什么机会

你可能不用Jira。但全球有几十万家企业用Jira管开发流程。当企业开始用Jira管AI agent，全新的服务需求就出现了。

第一个方向，帮企业搭建AI Agent工作流。大多数企业的Jira管理员是传统项目经理，对Claude Code、Cursor这些完全陌生。反过来熟悉这些agent工具的开发者，又不一定懂Jira配置。这个「翻译层」就是你的生意。一次配置收几千到一万，你说有没有市场。

第二个方向，Agent规格说明书代写。你没看错，这是一个新工种。很多企业的需求本身就模糊，扔给AI agent肯定做不对。有人能把「老板想要一个下单页面」翻译成「用React写、对接Stripe支付、包含表单验证的H5页面」这种agent能精确执行的东西。这种能力，说真的没几个人有，你能做就是稀缺。

第三个方向，AI开发效率审计服务。这个方向利润最高。DX的AI成本管理工具能算出每张ticket花了多少钱。你帮企业分析这些数据，哪些活该给agent、哪些必须人做。一个月一次报告收两三千，企业乐意掏，因为省回来的token费比这个多。

AI coding agent从人手头的工具，变成了企业管理的基础设施。就像当年的云服务器从个人折腾变成了AWS一样的完整服务体系。每一次基础设施级别的跃迁，都会带出一批新的服务商。

常见问题

Jira的AI Agent开发功能怎么使用？

在Jira中创建任务后，可以将任务分配给Claude Code、Cursor或GitHub Copilot等AI coding agent。通过Jira的Agent Sessions面板可以监控agent的工作进度，Agentic Engineering项目模板帮助你快速搭建对接框架。目前这些功能已在Jira Cloud中上线，付费计划均可使用。

AI Agent代写代码怎么收费？

如果你提供AI agent工作流配置服务，建议按项目收费，基础配置（Jira＋agent对接）3000到5000元，全流程搭建（含自动化规则、成本监控）8000到15000元。Agent规格说明书代写每份500到2000元，或按月2000到5000元。效率审计建议按月订阅，3000到8000元／月。

不懂编程也能做AI Agent开发服务吗？

至少需要基本的技术理解。你不一定自己写代码，但要理解AI coding agent能做什么、Jira的工作流怎么配。建议从最简单的方向入手，帮小型团队配置Jira Coding Agent做自动化bug修复，门槛最低。
