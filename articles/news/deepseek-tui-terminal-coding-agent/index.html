我在终端里用DeepSeek写代码，结果被它的"性格"给整懵了

2026-05-09阅读大约 6 分钟

事情是这样的。

昨天晚上刷GitHub Trending，突然被一个项目吸引了目光——DeepSeek-TUI，三天时间，3,731个star，直接冲到了Trending第一。

3,731个star是什么概念呢，就是你睡一觉醒来，它已经凉了，不对，是已经起飞了。

我当时的反应就是，它不就是个终端界面吗，DeepSeek网页版又不是不能用，为啥这么多人给它刷star？

结果装完之后，我沉默了。

黑暗森林。

先说安装。这玩意装起来意外地简单。

官方给的是pip install deepseek-tui，启动就一条命令，跟那些花里胡哨的编程助手比起来，这个安装流程简直清新脱俗。我当时还嘀咕，这么简单，该不会是个半成品吧。

结果打开之后，我愣住了。

它不是那种弹出一个对话框让你粘贴prompt的工具。它是直接在终端里跑了一个完整的交互界面，左边代码窗口，右边对话窗口，分屏的。

这意味着什么，你在VS Code里写代码的时候，旁边就蹲着一个DeepSeek随时伺候着。

不是复制粘贴，是直接在你的代码旁边开口说话。

我随便打开了一个之前写的Python脚本，想试试它的代码理解能力。我指着屏幕上的某一行问，这段逻辑有没有优化空间？

它几乎秒回，而且不是那种"这段代码可以更简洁"的废话，它直接给你重构方案，然后问你，要不要直接改？你说改，它啪一下给你改好了，diff都给你展示出来。

整个交互流畅得有点离谱，比网页版舒服多了。

网页版的DeepSeek我用过，遇到bug要复制粘贴上下文，有时候还要手动清理历史记录，交互割裂感很强。但这个TUI版本，它直接就在你代码边上，你问它答，改完直接看diff，体验很顺。

我试着让它帮我review了一段之前写的乱七八糟的脚本。

它不仅指出了问题，还顺着把相关的潜在风险都说了一遍，甚至给了一个小的优化建议。

怎么说呢，就是那种"你终于来了"的感觉。

不过话说回来，这个TUI跟网页版最大的区别，我觉得还不是速度或者准确率，而是那种"在场的陪伴感"。

网页版给我的感觉是你在跟一个服务器通信，但TUI版本，感觉是有人在隔壁工位跟你并肩作战。

这种感觉很难形容，但用过的应该懂。

当然，它不是银弹。

我试了一个比较复杂的并发场景问题，它给的方案理论上可行，但跑不起来。后来我想了想，这不是DeepSeek的问题，是我问的问题本身就有歧义，它顺着我的歧义往下走了。

其实就是，你得会问。

这个工具只是把DeepSeek的能力翻译成了一个更好用的形态，但它改变不了你提问的质量。

话说回来，3,731个star，Trending第一，我觉得已经说明问题了。

它解决的不是"DeepSeek能不能写代码"的问题，而是"DeepSeek写代码的时候我怎么才能不那么累"的问题。

这个需求，意外地真实。

对了，安装只需要一条命令。

pip install deepseek-tui

然后 deepseek-tui 启动就行了。

感兴趣的可以去GitHub看看这个项目，地址是 github.com/Hmbwn/DeepSeek-TUI，现在已经是Trending常客了。

说实话，用了这么多年编程辅助工具，这种"装完就忘不掉"的感觉，已经很久没有过了。

可能就是那种，就是单纯的，想看看现在的终端AI编程，能做到什么程度的冲动吧。

现在看来，答案是，能做到的事情，已经比我们以为的多很多了。
