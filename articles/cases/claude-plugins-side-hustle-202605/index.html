先说结论：这个机会为什么大？

很多人可能觉得，做插件不是要写代码吗？我一个普通人怎么做？

坦率的讲，放在一年前你说得对，开发一个VS Code扩展需要懂TypeScript、懂Extension API、懂调试打包发布，门槛确实不低。

但现在情况完全不一样了。你看看这个目录的结构就知道了。

一个Claude Code插件本质上就是一个带JSON配置文件的文件夹。你看它官方的文档怎么说，一个插件只需要这几样东西：一个.claude-plugin文件夹里面放个plugin.json，再加一个README.md。没了。

图片

plugins目录里面的internal插件和external_plugins目录里的第三方插件，结构几乎一模一样。这意味着什么？意味着Anthropic把插件开发的门槛降到了极低。

我自己的路径是，完全用Claude Code本身帮我写插件代码。我用自然语言跟Claude描述我想要的功能，它直接帮我把plugin.json和MCP配置都生成好了，我只需要测试一下能不能跑通，然后提个PR提交上去。

整个过程花了大概三个晚上，加起来不到6个小时，我发布了一个给Notion集成用的Claude Code插件。

我朋友说我这个路子太野了。但我觉得这恰恰说明了这个市场有多大。现在Claude Code已经发展了快一年，用户量巨大，但插件圈子才刚刚起步。你哪怕只会做最基础的文件操作插件、代码片段管理插件、定时任务插件，都有人买单。

这个目录到底长什么样

我仔细翻了一遍这个仓库，给你说说它的结构。

目录分成两大块。plugins目录里放的是Anthropic官方维护的插件，包括文件系统操作、终端执行、对话管理这些基础功能。这部分跟普通开发者没什么关系，我们重点看external_plugins。

external_plugins目前有15个第三方插件，你看看这些名字：Asana、Context7、Discord、FakeChat、Firebase、GitHub、GitLab、Greptile、iMessage、Laravel Boost、Linear、Playwright、Serena、Telegram、Terraform。

你这会发现，这些插件基本都是大厂或者知名开源项目的官方入驻。真正普通开发者做的插件还没形成规模，这就是窗口期。

图片

每个插件的结构跟官方文档说的一模一样，一个.claude-plugin目录加一个README。有些插件还带了.mcp.json的MCP服务器配置，但那是高级玩法，新手不用管那个。

怎么安装呢？直接在Claude Code终端敲 /plugin install 插件名@claude-plugins-official，或者在插件管理器里选Discover浏览。就这么简单。

怎么提交自己的插件呢？仓库里有个提交表单链接，你把插件的GitHub地址填上去，Anthropic团队审核通过就会合入。审核标准主要是安全和质量两方面，后面我会详细说。

赚钱的方式，我拆成三种模式

我研究了一圈，目前来看靠Claude Code插件赚钱主要有三条路，门槛从低到高。

第一种，卖插件配置教程加服务。这也是我目前在做的。很多人知道Claude Code好使，但配置插件、调参数、跟自己的工具链打通这些事儿，真的不想自己折腾。你花半小时帮他配置好，收个几十到几百块，这生意永远有人需要。

第二种，开发收费插件。虽然这个目录目前是开源的，但是，Anthropic没说不允许商业化。你完全可以开发一个解决特定痛点的插件，比如专门给小红书运营用的，内容生成插件，或者给设计师用的，样式检查插件。考虑到Claude Code的用户群体全是开发者，一个好用的专用插件定个49、99完全有人买。

第三种，接定制开发项目。这是单价最高的模式。哪个公司或者团队想做自己的内部工具集成，就找你定制一个Claude Code插件。我之前看到的那个例子，有人专门帮律所做案卷检索Agent，一个项目就收了12000。换成Claude Code插件定制，小一点的项目3000起步，大型项目1到3万很正常。

三种模式可以叠加做。我的计划是先把配置服务跑起来积累口碑，同时开发一个自己的收费插件，等单子多了再招兼职来分担交付。

具体怎么操作，我一步步说

如果你也想试试这条路，我手把手教你。

第一步，搞清楚Claude Code插件到底是个啥。

实际上你不用搞懂技术细节，只需要知道一件事。一个插件就是一个文件夹，里面有一个plugin.json文件描述这个插件叫什么、干什么用、作者是谁。就这么简单。

官方文档在 code.claude.com/docs/en/plugins，但说真的你连文档都不用通读，直接找一个existing的external插件fork下来改改就行。

第二步，找灵感，决定做什么。

最有效的方式是去看GitHub Issues。这个仓库有680个issues，全部是用户提的需求，有人想要WebSocket支持，有人想要Slack集成，有人想要AI代码审查插件。你看哪个需求提的人多、但你发现还没人做，那就是你的机会。

另外你也可以去一些开发社区，看看大家在Claude Code里最头疼什么。我自己从Notion切入，就是因为看到好几个开发群都在问Claude Code能不能直接读写Notion数据库的。

第三步，用AI帮你写插件。

这一步是降维打击。你不需要会TypeScript或者Python，因为Claude本身就懂。打开Claude Code，直接说我想做一个插件，功能是XXX，结构参考external_plugins里某个现成的插件。

Claude会帮你生成整个目录结构和plugin.json。你检查一下没问题，自己拉到本地跑一遍测试，就完事了。

第四步，提交到官方目录。

把插件放在GitHub上公开仓库，用那个提交表单（clau.de/plugin-directory-submission）提交链接。Anthropic团队会审核，主要是检查安全问题，比如插件有没有访问不该访问的文件、有没有网络请求会泄漏信息。大概一到两周会有结果。

在等待审核的过程中，你已经可以先赚钱了。把插件仓库链接发到知乎、掘金、V2EX这些地方，自然会有人来找你问能不能定制。

第五步，开始接单。

定价我建议你这样。配置服务类的，单个插件配置50到100。定制开发类的，按复杂度，简单的一两天能搞定的300到800，需要对接API和复杂逻辑的1000到3000。如果涉及到企业内部系统的集成，直接报5000以上，因为售后成本高。

我踩过的几个坑

说几个血的教训。

一个是plugin.json的格式。看似简单的JSON，但里面的name字段不能随便起，必须是全小写加连字符，不然Anthropic的审核机器人会直接打回。我第一版就是在这个细节上被卡了两天。

还有一个是MCP配置的问题。大部分插件不需要MCP，但如果你要做外部API调用类的插件，就必须配.mcp.json。这个配置稍微复杂一点，但网上很多现成的模板，直接改改就能用。

最后一个是定价的坑。开始我单子报得便宜了，199一个。后来发现一个配置加调试经常花两三个小时，算下来时薪还不到70，跟我预期差太远。后来调整到299起，复杂单子上到800，才算勉强合理。所以一开始别报太低，你提供的是省时间的价值，不是体力劳动。

一些更野的玩法

我看了一轮这个插件目录，还有一些思路我觉得值得试试。

一是做行业垂直插件。现在所有插件都是通用开发工具类的，没有人做过针对特定行业的插件。比如给数据分析师的SQL查询优化插件、给产品经理的需求文档管理插件、给运维人员的Kubernetes日志分析插件。这些垂直场景的插件需求真实，而且一旦占领了某个细分市场，别人很难抄。

二是做插件合集包。单个插件的功能有限，但如果你把10个常用的小工具打包成一个合集，对于用户来说是极大的便利。你可以收合集费，或者做成订阅制，每个月更新新的小工具。

三是做培训。市面上现在根本没有完整的Claude Code插件开发教程。你把自己开发第一个插件的过程录成视频、写成教程，不管是卖课还是放在知识星球里做付费内容，都有很强的需求。

说实话我觉得这个方向还处于非常早期的阶段。你知道最早做WordPress主题的那批人赚了多少吗？最早做Shopify App的那批人呢？每个平台的红利期都不一样，但规律是一样的，先进场的人吃大肉。

Claude Code的插件目录就是下一个这样的平台。你不一定非要是程序员才能做，因为AI已经帮你把写代码这件事干掉了。你需要的是发现需求的能力和执行力。

我一直觉得AI时代最大的机会不是学会AI本身，而是学会利用AI帮你完成以前做不了的事。开发插件这件事，放在一年前我要学两周才能上手，现在一个下午就能搞定。
