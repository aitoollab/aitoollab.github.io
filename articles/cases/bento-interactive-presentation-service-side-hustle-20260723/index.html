为什么交互式HTML演示是个好生意

先说需求。

传统PPT的市场有多大不用我说了。全球每天有超过3000万场演示在做。但2026年的情况是，越来越多的科技公司和创始人在抛弃PPT。不是因为PPT不好用，而是因为他们想要的东西PPT给不了。

比如实时数据仪表盘。比如内嵌一个可交互的产品demo。比如观众可以直接操作的图表。比如从点击到画面切换的无缝morph动画。

我跟你说，这些东西在PPT里要么做不到，要么做起来成本高到离谱。但在HTML里，这就是原生能力。

Bento解决了什么问题？它把「开发一个HTML演示文稿」这件事的门槛降到了几乎为零。你不需要懂React、不需要懂前端工程化，只需要会跟Claude Code对话。但即使有了Bento，大多数人还是不会自己去写。

为什么？因为科技公司的创始人没时间。你想想，一个创始人每天要见投资人、盯产品、管团队，他愿意花3000-8000元请人做一个惊艳的演示文稿，而不愿意花一整天自己跟Claude Code较劲。

这就是你的服务价值。

HN上已经有人在这么做了。评论区里好几个用户提到他们在用Claude Code + Bento做演示文稿，效果远超PPT。还有人提到用Slidev、Typst等替代方案。整个市场在快速升温。

三档服务定价，从入门到高阶

我设计了三个层级，对应不同客户的需求和预算。

第一档，基础版（3000-4000元）。

客户给你一个PPT或一份提纲和素材，你用Claude Code + Bento把它转成交互式HTML演示文稿。15-20页，包含基本的morph动画、页面切换、图文排版。客户拿到后可以自己在浏览器里编辑修改。

适合谁？科技公司内部汇报、学术会议演讲、产品发布前的预演版本。工作量大概2-3天。

第二档，进阶版（5000-6000元）。

在基础版之上，加入内嵌的实时数据图表（Bento自带live chart功能，hover有tooltip，可以缩放）、自定义交互元素、多场景切换。以及跟客户的品牌视觉完全一致的配色和字体方案。

适合谁？创始人做VC路演、投资人做LP分享、CTO做技术大会演讲。这些东西在传统PPT里做不出来，在Bento里就是原生能力。工作量3-5天。

第三档，高阶版（7000-8000元）。

在进阶版之上，加入内嵌的产品demo或数据看板、多语言版本自动切换、自定义动画路径、嵌入外部页面/API数据。最终的演示文稿是一个可以独立运行、永久归档、任何浏览器都能打开的完整HTML文件。

适合谁？大型产品的发布会、VC的年会演示、顶级技术大会的主题演讲。这种项目需要跟客户深度沟通，理解他们的叙事结构和演示场景。工作量5-7天。

工具链，你只需要这几样东西

这个服务的工具链非常轻。

核心工具，Claude Code。这是你的主力。其实吧，Claude Code可以直接在你的项目目录里写代码、建文件、做修改。你只需要告诉它你要什么样的演示文稿，它会帮你生成Bento格式的HTML骨架，然后你在此基础上修改内容。

Bento本身有Claude Code的skill配置。在它的官网上专门有一个页面教你给Claude Code装上Bento skill，之后Claude Code就能直接生成Bento格式的演示文稿了。

框架，Bento。你不需要每次都从零开发。Bento提供了一个560KB的样板文件，包含编辑器、查看器、所有功能。你在此基础上填充内容就行了。

辅助工具，图片处理用Canva或Figma，图标用Iconify或者Lucide，字体用Google Fonts。如果你需要协作用户反馈，直接发一个Bento文件过去就行，任何浏览器都能打开。

定价汇总，Claude Code的Pro Plan是$20/月（够了），Bento完全免费开源（MIT协议），Figma有免费版。前期总工具投入不到50美元。

完整操作流程，第一步到第七步

第一步，装上Bento的Claude Code skill。

去bento.page的官网，找到「Get Skill」按钮，复制配置文件放到Claude Code的skills目录下。这一步做完之后，你直接跟Claude Code说「帮我创建一个10页的Bento演示文稿，内容是XX产品的市场分析」，它就能直接生成。

第二步，用一个项目练手。

不要上来就接单。先用自己的东西练一次。可以是你的个人博客、你的side project、或者任何你熟悉的内容。目标是跑通「Claude Code生成 → Bento编辑微调 → 浏览器预览 → 效果满意」这个闭环。

Bento的编辑体验很直观，按下Esc键就能切换到编辑器模式，拖拽修改排版，双击修改文字，跟PPT的操作逻辑差不多。

第三步，建立自己的作品集。

做了两三个练手项目之后，挑最好的两个做成作品集。记住，你的客户是科技创始人、VC这些人，他们看的是视觉冲击力和交互流畅度。不要拿一个只有文字和图片的「换皮PPT」去展示，要拿那些有morph动画、live chart、交互元素的案例。

第四步，找你的第一批客户。

最佳渠道是创业者社群和科技社区。我整理了几条最有效的，

在推特/X上搜「need a deck for my pitch」「looking for presentation help」，这些都是明确有需求的信号。主动私信发一段自我介绍和作品集链接。成功率不算高，但胜在精准。

在小红书/知乎上发Bento演示效果的视频。录一段你做的演示文稿的slideshow模式，展示morph动画和交互效果，配文字「帮创始人做VC路演HTML演示文稿」。这种内容在小红书上很容易引发讨论，因为大多数人都没见过这种形式的演示。

在创业者微信群/知识星球里发一个「免费帮一位创始人做演示文稿」的抽奖活动。免费做一单，做好之后让客户帮你发朋友圈推荐。付出的时间成本就是最低的获客成本。

给科技类自媒体/博主私信。有些博主每周都要做线上分享或线下活动演讲，他们的演示需求很高但对形式有追求。你帮他们做一个惊艳的HTML演示，他们的观众看完会在评论区问「你这演示怎么做的」。这就是口碑传播的开始。

第五步，接单和确认需求。

跟客户第一次沟通时，确认三个东西，演示场景（路演/产品发布/内部分享）、目标观众（VC/客户/团队）、核心要传达的信息（1-3句话概括）。然后问客户要已有的素材，PPT、Keynote、Notion文档、或者就是一段语音描述都行。

我最推荐的方式是，让客户把要讲的内容录一段语音发过来，你听完把结构整理出来发回去确认。这样既省了客户的时间，也让开场文案更贴近客户的真实表达风格。

第六步，交付和修改。

Bento的好处是，交付的就是一个HTML文件。你发给客户，他在浏览器里打开就能用。如果有修改意见，Bento的编辑模式让他自己也能改。或者你修改后重新发一个文件就行。

我建议包含两次免费修改，超出后按500元/次收费。这样既保证了服务质量，也防止了无止境的修改。

第七步，维护和复购。

很多创始人隔两个月就要更新一轮融资材料。如果你第一次做得足够好，他会回来找你做第二版。你也可以提供「月度更新维护」套餐，每月1500-2000元，包含数据图表更新、内容调整。

踩过的坑，提前跟你说

我虽然没亲自做过这个方向，但HN评论区和Bento的GitHub issue里已经有人踩过这些坑了。

第一个坑，不要试图用Bento做超复杂的动画。

Bento的morph动画很惊艳，但它不是After Effects。如果你想做像素级的timeline控制或者复杂的粒子效果，Bento不是合适的工具。它的优势在于「足够的视觉冲击力但不需要开发成本」。管理好客户的预期很重要，上来就说「我要一个Apple发布会级别的动画」的客户，要小心接。

第二个坑，兼容性问题。

HN评论区有人提到在Firefox上运行Bento的动画会卡顿。Bento在Chrome上表现最好，在Firefox和Safari上偶有问题。交付前在至少两个浏览器上测试一遍。

第三个坑，签名式命名风险。

有个叫BentoPDF的免费PDF工具已经存在了，两者的名字容易混淆。如果你在内容中提到「Bento」，最好每次都说全称「Bento/Slides」来避免混淆。

第四个坑，跟现有PPT的结合。

有些客户会说「我的PPT已经做好了，你帮我转成HTML」。如果你直接用Claude Code去转换一个几百页的PPT，结果可能很糟糕。正确的做法是，抓取PPT的核心内容和结构，重新在Bento里构建，而不是试图做格式转换。

第五个坑，定价不要过低。

这个服务的本质是「帮客户省时间」，而客户的时间是很贵的。一个创始人自己花一整天跟Claude Code较劲，他的时间成本可能已经超过5000元了。你收3000-8000元帮他省掉这一天，他觉得很值。如果你收300元，他反而会怀疑你的质量。

常见问题

不会写代码能做吗？

能做。Claude Code负责生成代码，Bento负责可视化编辑。你只需要会用Claude Code对话即可。最简单的入门方式，去bento.page装上Claude Code skill，然后说「帮我创建一个10页的Bento演示文稿」，它就帮你生成了。

一单到底能赚多少钱？

基础版3000-4000元（15-20页，2-3天），进阶版5000-6000元（含动态图表和交互元素，3-5天），高阶版7000-8000元（含内嵌Demo和复杂动画，5-7天）。一个月接4-6单，月收入1.5-4万元。如果加月度维护费，还能更高。

跟用Gamma/Beautiful.ai做PPT有什么区别？

完全不同的东西。Gamma做的是传统PPT在线编辑，输出的是PPT文件。Bento做的是交互式HTML演示文稿，输出的是单个HTML文件，可以在浏览器里直接播放，支持morph动画、live chart、交互元素、内嵌Demo。目标客户也不同，Gamma面向普通商务人士，Bento面向科技公司创始人和VC。

需要哪些工具？每月成本多少？

Claude Code Pro $20/月，Bento免费开源，Figma免费版够用，Google Fonts免费。前期总投入不到50美元。客户只需要一个浏览器就能打开和编辑你交付的HTML文件。

怎么保证交付质量？

关键是在Chrome和Firefox两个浏览器上各测试一遍。制作品质基准（Bento的Demo页面本身就是最好的品质参考）。包含两次免费修改，超出后按500元/次收费。
