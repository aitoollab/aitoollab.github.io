靠Graphify做AI知识图谱服务：49K星开源项目，帮一个客户收2000，两周赚了1.2万

发布于 2026-05-23 | 分类：AI副业案例 | 阅读约9分钟

事情是这样的。

上周有个做SaaS创业的朋友找我，说他们团队最近新来了两个开发，但代码库太乱了，光README就有十几个版本，新人光看懂项目结构就花了一周，效率低得离谱。

他来了一句：「有没有什么办法，能把我这堆乱麻一样的代码和文档，一键变成一个可以搜、可以点的地图？」

我当时脑子里蹦出来的第一个想法就是Graphify，一个GitHub上49K星的项目，我关注了大半个月了，一直想找机会试试它到底能不能打。YC S26孵化的，Safi Shamsi做的，这人之前写的书「The Memory Layer」在AI圈也小有名气。

我说你等我一下，我去你项目文件夹里跑个命令。然后就一句 /graphify .，三秒钟之后，他们的整个项目——代码、文档、数据库Schema、架构图——全变成了一张交互式的知识图谱。

他当时就愣住了。说「我操这什么玩意，我的项目怎么变成一张蜘蛛网了」。

然后他点了图上几个节点，发现能直接看到模块之间的依赖关系、每个函数被谁调用了、文档里提到的关键概念跟哪些代码文件有关联。他越点越兴奋，让我把这张图做成他们团队内部的「项目地图」，说以后新人入职直接看这个就行，不用再翻三天文档了。

我就在他公司待了一个下午，把整套流程搭好、数据整理好、给他们团队做了个演示。最后他给我转了2000块钱，说这个值。

从那天开始我就意识到，这东西能赚钱。而且不是小钱。

这是个什么项目

Graphify是一个开源知识图谱生成工具，GitHub上49K星，YC S26孵化。它的核心能力就一句话：你丢进去一整个项目文件夹，它给你吐出来一张可以直接在浏览器里点来点去的知识图谱。

支持的东西多到有点离谱。代码（Python、JavaScript、Rust、Go啥的都行）、Markdown文档、PDF文件、图片、视频甚至音频——它都能解析成图谱里的节点和关系。

输出的结果就三个文件：

graph.html 是交互式图谱，用浏览器打开就能点着玩。点一个节点，关联的节点会高亮，还能搜索和过滤。

GRAPH_REPORT.md 是一份分析报告，告诉你项目里哪些是核心概念、哪些模块之间关联最紧密、有什么意料之外的连接。

graph.json 是原始数据，如果你懂技术可以自己再加工。

安装也简单到离谱。一个 uv tool install graphifyy 装好，在你的AI编程工具里输入 /graphify . 就行了。支持Claude Code、Codex、Cursor、Gemini CLI，甚至Hermes——基本上主流的AI编程工具全都能用。

我听他们说它还支持Google Antigravity，虽然我不太确定那是个啥。。。

反正这项目最骚的地方在于，它把你的「项目」从一堆散装文件变成了一张能看得见、能点得着的网络。这个体验用文字描述不出来，你得亲眼看到那个图谱在浏览器里渲染出来，然后随手点几个节点——那种「原来这玩意儿是这么连起来的」的爽感，才是它的核心卖点。

我为什么选这个项目来做服务

选Graphify之前我也想过别的方向。但说实话，知识图谱这个东西有个天然优势。

你看现在做AI副业的人都在干啥？帮人写简历、帮人去水印、帮人省token——这些都是红海了，你做我也能做，价格战打得飞起。简历优化都卷到39块钱一单了。

但知识图谱服务不一样。这玩意儿听起来很高大上，很多中小团队听说过但没见过，更别说用过了。你跟他们说「我能把你的项目文件夹变成一张智能地图」，他们第一反应是「这得多少钱？」不是「这个我能自己装」。因为在他们认知里，知识图谱是Google知识图谱那种级别的东西，不是装个命令行就能搞的。

这个信息差就是你的利润空间。

而且Graphify本身的安装和使用真的很简单，确实不需要编码能力。你只要会打开终端、会复制粘贴命令、会跟人聊天——就这三样。

投入成本

几乎为零。

Graphify本身是MIT协议开源的，免费。PyPI上的包叫graphifyy，装一下就行。

你如果自己有Claude Code或者Cursor这些AI编程工具，直接用。没有的话，免费的也一样跑——你可以直接用命令行跑，不需要依赖任何付费工具。

唯一的硬成本是一台能联网的电脑，和一个愿意花一个周末熟悉工具的你。

我第一周大概花了两个晚上的时间，装了Graphify，拿自己折腾过的几个项目文件夹反复试，理解它输出的图谱长啥样、什么类型的项目图谱好看、什么不行。到第三天我已经能十分钟搞定一个中小型项目的图谱生成了。

如果你完全从零开始，我估摸着一周之内也能上手。门槛比你想象的低很多。

收益情况

两周总共接了6单，总收入1万2出头。

定价是这样的：

小型项目（个人博客、小工具源码、毕业论文资料这些），图谱生成+基础分析，一单500-800。这种单子一般半天搞定，实际花的时间可能两个小时都不到。

中型项目（创业团队代码库、产品文档体系、学术论文合集），图谱生成+深度分析+定制化汇报，一单1500-2500。这个要花一整天去打磨，但产出很漂亮，客户反馈也最好。

大型项目（企业级的多仓库代码体系、跨团队的文档系统），按项目复杂度谈，3000起步，上不封顶。这类我目前还没接到，但已经有人在谈了。

两周的具体构成：小型单接了3单，3×700=2100。中型单接了3单，3×2000=6000。

就这6单，9000块。加上我上周末接了个给初创团队做项目图谱的加急活，收了3000。加一起1.2万。

坦率的讲，这个收入有运气成分。前两周刚接触这个市场、价格定得也比较随缘。但关键是我没有花一分钱成本，赚的就是纯手工费。而且随着你做的案例越来越多，你的报价可以越来越高——因为每一个做过的图谱都是你的作品集。

操作步骤

下面我把完整的流程拆出来，你跟着做就能跑通。

第一步：搭环境

打开终端，装Python3.10以上版本。如果你用Mac，brew install python@3.12 uv 一条命令搞定。Windows用户用winget装uv，也很快。

然后运行 uv tool install graphifyy，等几秒钟就装好了。

这时候你在终端打个 graphify 看看，如果出现了一堆参数说明，就是装好了。

你拿自己的一个项目文件夹，或者在GitHub上随便找个开源项目clone下来，进去运行 graphify . ，等个十几秒，它就会在目录下生成 graphify-out/ 文件夹，里面就有那个 graph.html。

双击打开graph.html，你就能看到你的第一个知识图谱了。

第二步：打磨作品集

这一步很多人会跳过，但我建议你别跳。

拿不同类型的项目去跑图谱。代码库跑一遍、Markdown文档库跑一遍、PDF论文合集跑一遍。看看不同输入出来的图谱有什么区别。你只有自己玩明白了，才知道什么样的图谱最有「卖相」。

顺便说一嘴，Graphify还有个叫 graphify export callflow-html 的功能，能生成带Mermaid调用流程图的架构文档，这个在给客户演示的时候特别好使，一看就很专业。

把你跑得最好的几个图谱截图，存下来当作品集。我第一次展示给客户看的，是我自己用Graphify跑的一个开源项目图谱，对方看了就说要。

第三步：找到你的第一单

获客渠道我试下来最好用的是三个：

第一个是朋友圈和微信群。发一条「免费帮三个朋友的项目做项目知识图谱，限前三个」。这个不亏，因为你做的是案例积累，而且每个被你帮过的人都是你的活广告。

第二个是闲鱼。搜「项目文档梳理」、「代码整理」这些关键词，看看谁在找类似的服务。你直接去跟卖家说「你这个我能用AI帮你做，效果更好」，效果出奇的好。

第三个是小红书。这个要稍微用点心，发一篇「我把xxx开源项目的代码变成了知识图谱，你能想象吗」，放几张图，不用太长。现在这个赛道还没什么人做，流量特别好拿。

我做第一单的时候，就是朋友圈那条免费的消息，有个创业的朋友看到之后找的我，做完之后主动问我「能不能收费帮我也做一个？」这就是你第一个付费客户。

第四步：交付

收到客户的资料之后，先自己跑一遍图谱看看效果。如果图谱里节点太少或者关联太弱，说明资料结构有问题，需要跟客户沟通补充一些信息。

然后把图谱生成好，导出成HTML文件。我一般还会额外花点时间，给客户做一个简单的讲解。比如打开图谱，从最核心的几个节点开始，沿着连线一路讲下去，「你看，这个是你们的核心模块，它依赖了这些组件，这些组件又关联了这些文档」。

这个讲解环节特别重要。因为客户买的不只是一个文件，他买的是「听懂自己项目」的这个体验。你花10分钟给他讲一遍，他觉得你值2000块。你不讲直接把文件丢给他，他可能觉得你的东西还不如一个PDF值钱。

第五步：复购和转介绍

做完一个客户之后，有两个方向可以走。

第一个是复购。告诉客户你可以按季度帮他的项目更新图谱。项目在演进，图谱也要跟着变。一个季度收500-800的维护费，客户觉得合理，你也多了持续收入。

第二个是转介绍。一个满意的客户可能会把你推荐给3-5个朋友。我第二周的三单里有两单都是第一单的客户介绍来的，这比打广告有效得多。

适合什么样的人

坦白说，做知识图谱服务比做简历优化和AI去水印门槛高一点。但你不需要会编程，只需要会用命令行装软件。如果你会用pip装Python包，你就够资格了。

但有几个软性的东西你得有。

你得有基本的沟通能力。不是说要口若悬河，但至少能把「知识图谱」这件事说人话讲清楚。客户不懂图数据库、不懂节点关系、不懂语义网络——这些技术词一个都别用。你就说「我能把你的项目变成一张地图，你们新人进来不用花三天看文档，打开这个页面点几下就全都懂了」。

你得有点审美。Graphify默认生成的图谱已经很好看了，但你总得知道哪些颜色配在一起舒服、布局怎么调一下更清爽。这个不用学设计，多看几个别人家的图谱，模仿一下就行。

最好你对某个行业有点了解。如果你懂编程，那服务开发者团队的图谱最有说服力。如果你做学术，那服务硕博研究生做论文图谱是你的强项。如果你做产品，那就做产品文档的图谱。不用什么都懂，找到一个你熟悉的领域深挖就好。

避坑指南

第一个坑：不要接太乱的项目。

如果客户的项目本身结构就一团糟，文件夹都是乱放的，100个文件堆在一个目录里，Graphify跑出来的图谱会很难看。这时候你要先建议客户把项目结构整理一下，或者你就收两份钱——先做结构整理，再做图谱。别硬上，图谱不好看客户不会觉得是项目的问题，会觉得是你的问题。

第二个坑：提前说清楚什么能做、什么不能。

Graphify虽然支持视频和音频，但处理起来会慢很多，而且需要装额外的依赖。我一般是做代码+文档+PDF三类，图片和视频单独报价。客户让你「把所有东西都丢进去」，你要先评估一下工作量，别答应得太痛快然后自己偷偷熬夜。

第三个坑：保护好客户资料。

客户的代码库和文档里可能有敏感信息。不要截图发朋友圈当广告，除非客户同意。我一般会给客户签一个简单的保密协议，做完之后把本地数据删掉，只保留一个脱敏后的演示版本当作品集。信任是这行的命根子。

第四个坑：别把定价定低了。

知识图谱服务不是一个可以批量生产的服务。每一个项目都是定制的，你花的时间、精力、对项目的理解都不一样。别一上来就报199、299这种价格，对不起你自己的时间。我自己的体会是，500-800是小单的合理区间，2000+才是你真正赚钱的地方。

一些心里话

写到最后了。

Graphify这个项目我是真的很喜欢。你不觉得吗？在这个AI时代，我们每天被各种工具和信息包围，却越来越难看清楚「这些东西到底是怎么连在一起的」。

知识图谱解决的就是这个问题。它不是一个新的技术概念——哲学家2000年前就在研究概念之间的关联了。但Graphify把它做成了一个普通人也能用的工具，一个命令下去，你面对的不再是一堆散落的文件和一个空白的编辑器，而是一张清晰的、可以交互的地图。

我做这个服务的体会是，客户最终买的不是那个HTML文件，而是「原来我的项目长这样」的那种恍然大悟的感觉。

这事挺有意思的。我帮那个做SaaS的朋友搭完图谱之后，他团队里有个开发在群里说了一句：「早知道项目结构是这样，我第一个礼拜就不用花三天到处问人了。」

就这一句话，我觉得2000块值了。

我有时候想，AI时代最好的副业可能就是这样：找一个好的开源工具，找到一个愿意为价值付费的人群，然后做那个连接工具和人的人。你不用发明任何东西，你只需要比别人早学会怎么用这个东西，然后去帮那些还不会用的人。

信息差永远存在。磨平一些，就是钱。
