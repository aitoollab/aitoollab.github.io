用4万星开源AI Agent帮小企业做自动化，月入2万实操拆解

2026年6月7日 · AiToollab · AI副业赚钱案例

上周有个做电商的朋友给我发消息，说他招的客服又跑了。第三个月了，已经是第四个。

他做的是那种跨境小铺子，卖家居用品，一天大概200来个客户咨询。量不大，但问题杂，什么物流时效、尺寸换算、退换货规则，得有人一直盯着回复。之前请的大学生兼职，干了两个月嫌工资低走了。后来找了外包客服，一周后发现回复质量一塌糊涂，客户投诉好几个。

他跟我说，真的，哪怕有个东西能自动回80%的常见问题，剩下的我亲自处理都行。

我当时第一反应不是「你去买个AI客服SaaS啊」，而是想起了前两天在GitHub上刷到的一个项目。

Nanobot，43000多星，香港大学数据科学实验室开源的。一个超轻量的AI Agent框架，能直接接入Telegram、飞书、钉钉、Slack、Discord、Teams、微信、邮件……十几个平台。记忆系统、MCP工具、模型路由、定时任务，全内置了。

我当天晚上就把Nanobot装起来试了一下，然后给他那个店铺搭了一套AI客服。

三天后他给我转了2000块，跟我说，「你帮我把这个装到我另一个店铺里去。」

这就是这篇文章要讲的事。

为什么Nanobot这个方向能赚钱

我先说一下我对这个项目的判断。

市面上的AI客服SaaS产品不少，智齿、网易七鱼、乐言这些，功能都挺全。但它们有几个问题，对微型企业来说特别扎心。

贵。正经的AI客服SaaS，一个账号一年大几千到上万。对于一个日咨询量200的小店铺来说，这笔钱心疼。

重。要配知识库、训练意图、设定回复模板，折腾一周才能上线。小老板哪有这个时间。

死。大多数SaaS客服只能接网页和微信公众号，你的客户在Telegram上问、在WhatsApp上问，它接不住。

而Nanobot的特点是什么？免费，轻量，通吃。

开源免费，零软件成本。pip install nanobot-ai，一行命令装好。配好API key，选好模型，连上你想接的聊天平台，就能跑了。

它支持的平台列表长到离谱。Telegram、飞书、钉钉、Slack、Discord、Teams、Signal、Matrix、QQ、微信、WhatsApp、邮件……基本上你能想到的聊天工具都支持。而且不是那种粗糙的转发，是有线程管理的，Discord和飞书的thread回复它都能处理。

记忆系统用Dream实现，Agent能记住之前聊过的内容。模型路由可以配fallback，主力模型挂了自动切备用。MCP工具集成让Agent能调用外部能力，比如搜索网页、操作文件、调API。

我自己的判断是，对小企业来说，一个能直接连到他们正在用的聊天工具上的AI Agent，比任何SaaS客服都实用。

因为你不需要改变客户的行为习惯。客户在Telegram上问问题，AI在Telegram上直接回复。客户感知不到「系统」的存在，只觉得老板回复变快了。

说真的，很多人可能不知道，像nanobot这种开源项目的商业价值往往被严重低估。大部分人觉得「开源免费的东西怎么能赚钱」，但服务永远比软件贵，部署永远比产品值钱。

我帮奶茶店老板做了什么

我那个电商朋友的case比较直接，就是接Telegram做自动客服。但为了摸清这个方向的边界，我又多找了一个客户测试。

是一个开奶茶店的本地老板。三间连锁店，两家有外卖平台，一家纯堂食。他的痛点是，美团和饿了么上的消息回复经常来不及，尤其是高峰期，店员忙着出杯根本没空看手机。差评里有一半在骂「回复慢」。

另外他有一个福利群，200多人的微信群，经常有人在群里问「今天有什么新品」「XX店还有货吗」「能不能提前预定生日蛋糕」。这些重复问题每天要答几十遍。

我的方案是这样的。

帮他部署了一个Nanobot实例，跑在一台2核4G的轻量云上，一个月99块。配了DeepSeek的API做主力模型，便宜，token费一个月大概几十块就够。然后接了两个通道，一个是微信公众号的客服消息，一个是那个200人微信群（通过微信群bridge接的）。

知识库就一份Excel，他自己整理的FAQ。新品列表、门店地址营业时间、外卖配送范围、预定流程、价格表。我把这些导入Nanobot的记忆系统，设了三条规则。

1，能回答的问题直接回答，语气要像店员说话，不要机器味。
2，回答不了的问题，回复「我去问一下店长，稍等哈」，然后标记未处理。
3，涉及价格和库存的实时信息，调用外卖平台的API查，不要瞎猜。

整个部署调试花了我大概三个小时，主要时间花在调prompt让回答语气别那么僵硬，以及搞定微信群的bridge连接。

上线第一周，他自己统计了一下，微信群里的常见问题AI自动回了大概85%，剩下15%标记为未处理的他抽空回。美团上的客服消息回复率从之前的60%提升到了95%以上。

差评率直接降了一截。

这单我收了2000。

为啥比电商那个贵500？因为奶茶这个case多了微信群接入和prompt调优，多花了差不多一倍时间。而且老板要求语气必须像真人，不要让顾客觉得是机器人，这个调起来挺费劲的。

API费用，你自己测试的话一个月也就几十块。

服务器，2核4G轻量云99元/月起步，4核8G约198元/月。大部分小客户2核4G够用。

你赚多少

基础部署，单平台接入，1000-1500元/套。一个半小时到两小时能搞定。
进阶部署，多平台接入加prompt调优，2000-3000元/套。三到四小时。
月度维护费，500-1500元/月。处理小问题、调prompt、加新功能。
我自己这两周的数据，做了2单，电商朋友那单1500加奶茶店这单2000，一共3500。奶茶店签了月度维护，每月800块。同时电商朋友又介绍了一个做代购的在谈，那单大概率能成。

如果稳定下来，兼职做每月6-8单基础加2-3单进阶，月入8000-12000是稳的。维护合同签得越多越香，因为维护几乎不花时间，大部分就是看看日志、偶尔改改prompt。全职做的话，月入2万完全可以期待。

坦率的讲，利润率比我想的要高。因为你的核心成本就是时间，而时间成本会随着经验积累快速下降。第一单三个小时，第五单可能一个小时就搞定了。

操作步骤，从零到接单

如果你看完觉得这个方向可以试试，我把我自己走过的路捋一下。

第一步，把Nanobot自己跑起来

GitHub上搜HKUDS/nanobot，43000多星那个。安装很简单，pip install nanobot-ai就行，也可以用uv装。
装完之后跑nanobot onboard，它会引导你做初始化配置。然后在~/.nanobot/config.json里配好你的API key和模型。我用OpenRouter做路由，因为便宜，DeepSeek做主力、Claude Haiku做备选。
跑nanobot agent启动命令行聊天，测试基本功能。然后再开WebUI，浏览器访问8765端口。
把每个功能都过一遍，记忆系统、工具调用、定时任务、目标追踪。你自己不熟的话，客户一问就露馅。

第二步，选一个场景深度测试

别上来就想着做全平台全能型方案。先选一个你最熟悉的场景吃透。
比如你做电商的，就先测试电商客服场景。你自己注册一个Telegram Bot，配好Nanobot接上去，模拟客户提问，看回答质量。反复调prompt直到你觉得「这回答我要是客户我愿意买单」。
记住，调prompt是最花时间也最值钱的一步。同样的Nanobot，prompt调得好和调得差，效果天差地别。这也是你收费的底气所在。

调prompt的一个小技巧，不要写「你是一个专业的客服机器人」，客户一听就烦。写「你是XX店的店长小助手，说话要像店员一样亲切自然，遇到不确定的问题就说我帮您问一下」。语气设定比知识库还重要。

第三步，准备你的服务方案

我建议分两档。
基础版，单聊天平台接入，基础prompt配置，一小时培训。收费1000-1500。
进阶版，多平台接入，定制prompt调优，记忆系统配置，月度维护。收费2000-3000加维护费。
前期只推基础版，先拿案例。有了三五个成功案例之后，再推进阶版和企业定制。

第四步，找客户

目标客户画像很清晰，有重复性客服需求但请不起专职客服的微型企业。
跨境电商小卖家，日咨询50-300单的。
本地餐饮和零售，有外卖平台和微信群的。
小工作室和培训机构，需要自动回复咨询的。
自媒体团队，需要AI帮忙管社群的。
怎么找？58同城和闲鱼搜「客服」「代回复」，小红书搜「小店铺运营」「一个人开店」，能看到大量有这类需求的老板。或者混进本地商家微信群，先潜水观察谁在抱怨客服问题。
不要直接上来就推销Nanobot。先问他们现在怎么处理客服，哪里最头疼。他们自己会说出来的。然后你顺水推舟，说我能帮你搞一个自动回复的AI，直接接到你微信/Telegram上，试问你感不感兴趣。

第五步，把经验标准化

每做一单就记笔记。部署步骤、常见问题、prompt模板、配置参数。三单之后你就能整理出一套标准化流程。
做成模板脚本之后，新单子的交付时间会急剧缩短。从三个小时到一个小时到一个小时以内。
交付的时候录一个三分钟的操作视频给客户，教他怎么看日志、怎么重启服务、怎么改简单的配置。大部分小问题他就能自己处理了，减少你的售后负担。

这活适合什么人

先说门槛。

你需要会基本的Linux命令行。知道怎么SSH连服务器、装软件、看日志、改配置文件。不要求多深，但基本的操作要溜。

需要用过Docker或者至少了解Docker的概念。Nanobot可以用Docker部署也可以直接pip装，但Docker更干净，推荐。

需要知道AI API是什么，怎么配key，怎么选模型。如果你之前用过ChatGPT的API或者玩过Ollama，这块不是问题。

不需要会写代码。Nanobot的配置全是JSON文件，改参数就行了。跟帮人部署self-hosted AI工作区那类活比，技术门槛其实更低，因为Nanobot本身就设计得很轻量。

但有一点特别重要，也是很多人忽略的。你的客户不是技术人员，他们听不懂你说「你的MCP server端口冲突了」或者「你需要换一个模型provider」。你得翻译成人话，说「我帮你调了一下AI的回复速度，现在快多了」。

说到底你卖的不是技术，是解决问题的能力。

踩过的坑，提前知道能省你不少事

坑一，微信群接入比你想的麻烦

Nanobot支持微信，但微信的API限制特别多。需要通过第三方bridge来中转，bridge的稳定性参差不齐。我试了两个bridge，一个经常断连，一个延迟很高。最后选了一个比较稳定的，但还是要定期维护。
如果你接单涉及微信群，一定要提前跟客户说明白，微信通道的稳定性不如Telegram和飞书，偶尔会有消息丢失或延迟。管理好预期比什么都重要。

坑二，prompt调不好等于白装

这可能是新手最容易犯的错。把Nanobot装好、接上平台，觉得大功告成了。结果客户一试，AI回答要么太生硬要么答非所问，直接给你差评。
我跟你说，安装部署只占30%的工作量，70%在调prompt和配置。你得花时间了解客户的业务，知道哪些问题最常被问到，答案的标准格式是什么，语气应该是什么样的。
我给奶茶店调prompt就花了将近一个小时，反复让老板测试、反馈、再改。这个过程省不掉。

坑三，别给客户装最新版

Nanobot更新频率极高，几乎每天都有新功能。但最新版有时候有bug，我亲眼见过一次更新之后记忆系统抽风，Agent开始胡说八道。
给客户装一定要用稳定的release版本。你自己的测试环境可以追最新版，客户的production环境只装经过验证的版本。出了问题你负责，不是GitHub issue负责。

坑四，服务器安全不能偷懒

有些客户觉得「反正是小生意，安全问题无所谓」。但AI Agent接了他们的聊天平台之后，能读到所有客户消息。如果你的服务器被入侵了，这些数据就全泄露了。
最基本的操作，改掉所有默认密码，开防火墙只开放必要端口，不要用root运行，定期更新系统。
这些事做好了你不说客户也不知道，但做不好出了事你百口莫辩。

再说一个很重要的事，客户有时候会提出一些Nanobot目前还做不到的需求，比如「能不能自动下单」「能不能对接我的ERP系统」。这时候别硬上，诚实说这个目前做不到，但可以后续迭代加上。承诺了做不到，比一开始就说不行更败人品。

最后说几句

我写这篇文章的时候一直在想一个事儿。

几个月前AI Agent还是程序员圈子里的玩具，大家折腾各种Agent框架，看它能不能自动写代码、能不能做研究。但最近风向明显变了，Agent开始真正进入业务场景了。上周UBER宣布给员工每月1500美元的AI使用额度，说明大公司已经在认真对待Agent的效率价值。

但大公司有大公司的用法，小企业有小企业的需求。一个日咨询200单的小卖家，不需要什么复杂的Agent编排，他只需要一个能接到微信和Telegram上、自动回常见问题、偶尔调一下库存API的AI就够了。

Nanobot恰好能满足这个需求。43000多星，MIT协议，文档覆盖9种语言，社区活跃度很高。但知道它的普通人还很少，更不知道能用它来赚钱。

我觉得这个窗口期可能就几个月。等到AI Agent部署变得像装WordPress一样简单，这个方向就没那么好做了。但至少现在，需求是真的，供给是缺的，你能做，有人愿付。

我以前在另一篇文章里提过一个观点，AI时代最值钱的不是AI本身，是帮别人把AI用起来的能力。Nanobot这个方向就是这句话的一个具体注脚。

常见问题

用Nanobot帮小企业做AI自动化，到底能赚多少钱？

单次部署收费1000-3000元，月度维护费500-1500元。兼职做稳定下来月入5000-10000元，全职做月入1.5万-2.5万元。维护合同是持续收入来源，签得越多越稳定。

普通人不会编程，能做AI Agent部署服务吗？

需要会基本的Linux命令行操作，会用Docker，知道怎么配环境变量。不需要会写代码，Nanobot的配置都是JSON文件，改几个参数就行。完全零基础的话需要一到两周学习时间，建议先用一台云服务器自己练手。

Nanobot和ChatGPT有什么区别？适合什么场景？

ChatGPT是通用聊天工具，需要你主动去问。Nanobot是AI Agent，可以自动接入你的Telegram、飞书、钉钉、Slack等聊天平台，自动回复消息、管理日程、处理文件、定时提醒。适合需要把AI嵌入到日常工作流里的小企业和工作室。

帮人部署Nanobot需要多少成本投入？

Nanobot完全开源免费，零软件成本。云服务器最低2核4G约99元/月。API费用看客户使用量，用DeepSeek的话一般月消耗50-200元。你前期投入主要是学习时间和一台99元的测试服务器。

小企业AI自动化部署的客户从哪里找？

目标客户是有重复性工作需要自动化的微型企业，奶茶店、餐饮连锁、电商工作室、小培训机构、自媒体团队。去58同城、闲鱼、小红书搜本地商家，或者混进本地商家社群。先聊痛点再出方案，成交率远高于直接推销。
