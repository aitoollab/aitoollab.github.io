省Token就是省钱：穴居人说话法让我每月少花60%API费

昨晚看到 GitHub 上有个项目冲到 45k star，叫 caveman。名字挺搞笑的——"穴居人"。点进去一看，是个 Claude Code skill，号称能省 65% 的 Token。

我的第一反应是：又来，营销号吧。

但看了看具体的原理和评论区的实测数据，还真不是吹的。这个项目的核心思想就一句话：把你的提示词从"英文老师教你写作文"模式改成"老外打电话说半句话也能懂"模式。

65% Token怎么省出来的

先说个我自己的踩坑经历。

上个月我用 Claude API 做了一个副业项目，给一个小红书账号做内容生成。写了大概 200 篇笔记，月底账单出来——¥847。当时就傻了，我以为最多¥200。

问题出在哪？我看了下 API 用量，每条回复平均消耗 3000 多 Token。但实际上核心指令就那么几个词，我写了一堆 "Could you please help me generate..." "Please make sure it is engaging and..."

这些 "please" "make sure" "engaging" 加起来没多少内容，但 Token 一个没少占。

caveman 这个项目就是来解决这个问题的。它不是让你说"坏英文"，而是把英文提示词压缩到最小语义单元。AI 理解能力很强，你少说废话，它照样能懂。

对比一下

普通写法：

Could you please help me refactor this function to make it more efficient and easier to maintain?

穴居人写法：

improve function efficiency maintain

意思一模一样，Token 少了 80%。Claude 收到 "improve function efficiency maintain" 完全能理解你要干什么。

实测效果（我自己跑了一遍）

我花了一晚上把手上几个常用提示词改成穴居人风格，重跑了之前那个小红书项目。

结果：同样的 200 篇笔记，Token 消耗从 60 万降到 21 万。按 Claude 3.5 Sonnet 的价格算，每千 Token $0.003，从 ¥847 降到 ¥295。

一个月省了 ¥552，一年就是 ¥6624。够买两顿火锅了。

个人经验：省 Token 最明显的场景是高频短任务，比如生成标题、生成标签、写开场白。如果是复杂的长文任务，压缩效果会打折扣——但也还能省 30-40%。

三步上手穴居人说话法

不需要装任何东西，记住三个原则就行：

1. 删掉所有客气话

"please" "could you" "would you mind" "I would appreciate it if you could" ——全删。这些对 AI 来说就是噪音。

2. 名词动词直接拼

不要完整句子，把核心词串起来就行。

  • "Generate 5 catchy headlines for" → headlines 5
  • "Write an engaging opening paragraph about" → opening para
  • "Check if there are any bugs in this code" → find bugs

3. 介词能省就省

"Write a response to this email" → reply email

"Convert this table into JSON format" → table to JSON

一开始可能会不适应，觉得自己在虐待 AI。放心，Claude 不介意。

什么人适合用这个技巧

强烈推荐：每月 API 花费超过 ¥200 的副业玩家。省出来的钱是纯利润。

可以试试：用 Claude Code 写代码的同学。caveman 项目本来就是给 Claude Code 用的 skill，装上就能用。

用处不大：偶尔用用 ChatGPT 的人。省了也没多少，还影响使用体验。

注意：穴居人说话法主要针对英文提示词。如果你的工作流是中文的，压缩效果会打折扣——中文本身就是压缩语言，一个汉字能顶好几个英文单词。但把中文提示词里的"麻烦帮我""能不能请"删掉，也还是有效果的。

我的工作流变化

用了两周后，我现在写提示词分两步：

第一步：正常写，用自然语言把需求说清楚。

第二步：Ctrl+C → 压缩成穴居人版 → Ctrl+V 发给 Claude。

压缩这步熟练了大概 3 秒钟。用了两天就形成肌肉记忆了。

如果你正在做 AI 副业，API 成本是必须要控制的一环。相同的工作，省 50% 成本就是多 50% 利润。这个技巧不能让你多用 AI，但能让你花更少的钱用同样多的 AI。
