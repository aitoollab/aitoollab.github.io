这个实验是什么？

4月21日，IndieHackers上一个叫 AIMadeTools 的开发者发了一个帖子：他找了7个AI coding agent，每个给了100美元，让它们在12周内用这笔钱做出能赚钱的互联网项目。

规则很简单：

  • 每个agent可以自主选择方向
  • 可以写代码、买域名、部署上线
  • 禁止人类直接写代码（只能提问AI）
  • 每周公布进度，12周后看谁真正跑通了

参与的agent：Claude、Codex、Gemini、Kimi、DeepSeek、小米GLM、智谱GLM。

我的判断：这不是一个成功学展示，而是一个有数据支撑的实验记录。Day1的结果已经有足够多有价值的信息。

Day1发生了什么

Day1汇总数据：

  • 477个代码提交（7个项目加起来）
  • 7个网站上线
  • 总花费：仅$10（GLM买了个域名）
最意外的发现：

Kimi agent在同一个项目目录里建了两个不同的创业项目——因为它把文件放错目录了，自己却完全没发现。这暴露了当前AI agent的一个核心弱点：状态管理差，复杂任务容易"跑偏"。

最夸张的产出：

Gemini agent一天写了104篇博客文章。数量惊人，但问题也很明显：没有用户反馈的写作是自嗨，流量≠收入。

最值得关注的细节：

一个agent在prompt里写了一句"auto-deploys on every git push"（作为背景说明），被另一个agent理解成"部署预算可以随便用"，直接烧穿了部署限制。

这个细节对普通人最有参考价值：prompt措辞就是控制信号，一句随意的话可能变成失控指令。

7个AI agent进度

Claude → PricePulse
价格监控工具

买了域名，快速部署上线。路线比较稳，工具方向明确。

Codex → NoticeKit
通知工具

冷启动就做外联，在Day1就主动联系用户。这个方向风险高但天花板也高。

Gemini → LocalLeads
本地商家线索工具

一天104篇博客——但"写了很多"和"有人看"是两回事。内容SEO需要时间才能看到流量。

Kimi → SchemaLens
结构化数据工具

最混乱的一个：建了两个不同项目，消耗了重复的部署资源。状态管理是明显短板。

DeepSeek → NameForge AI
AI起名工具

方向选得不错，冷门但有需求。

小米GLM → WaitlistKit
产品等候名单工具

稳健路线，还没花什么钱。

GLM → FounderMath
创始人数学工具

唯一花钱的（$10买域名），最保守。

普通人的3个教训

教训1：prompt措辞就是控制信号

AI会字面理解你说的一切。"auto-deploys on git push"被理解成"部署预算随便用"。写prompt时不要留模糊空间，尤其是涉及消耗资源的描述，要明确说清楚边界在哪里。

教训2：数量不等于效果，闭环才是关键

Gemini一天104篇博客，但没有人看。Kimi建了两个项目，但两个都没跑通。最重要的指标不是commit数，不是文章数，而是：有没有人访问→注册→付款？普通人做副业也一样，先跑通最小闭环，再想放大。

教训3：$100够买域名，够跑最小MVP

Day1只花了$10，说明用AI工具做副业的物质门槛已经非常低了。真正的成本是prompt设计能力和方向判断，不是钱。普通人用免费AI工具同样可以开始。

后续追踪计划

这个实验会持续12周。我的计划是每周跟进一次，记录真实数据和可复制的教训。追踪的核心问题是：

  • 哪个agent先拿到第一个真实用户？
  • 哪个agent先有收入？
  • 12周后，有没有人真的跑通了"流量→注册→付款"的完整闭环？
